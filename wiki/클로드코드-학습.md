# 클로드코드 (Claude Code) — 학습 노트

> [!tip] 서포트팀 이사 관점 핵심 takeaway
> Claude Code는 **"시키면 진짜 다 만들어주는"** AI 코딩 도구다 — 이 위키([[인사정보시스템-연차관리]] 등 사내 시스템, [[머니포워드-회계시스템]] 도입 지원)를 Claude Code로 운영하고 있다는 점에서 이미 실무 적용 중. LLM Wiki 방식(카파시)의 핵심은 **raw 폴더 자료 → Claude Code → wiki 페이지** 흐름이며, 지식이 "복리로 쌓이는" 구조가 장점. 비용 절감을 위해 자주 쓰는 명령어는 치트시트로 관리하고, 플랜 모드·자동승인 등 중급 기능을 먼저 익히면 작업 속도가 크게 올라간다.

## 영상 2 — Claude Code 왕초보 입문 23가지 팁

🔗 [영상 보기](https://www.youtube.com/watch?v=1_bRmkUvjHA)
**채널**: AI싱크클럽 | **게시일**: 2026-02-11

### 핵심 요약
- [`0:19`](https://www.youtube.com/watch?v=1_bRmkUvjHA&t=19s) **Claude Code의 차별점** — Cursor/Copilot은 파일 단위 자동완성, Claude Code는 **프로젝트 전체 이해 + 파일 동시 수정 + 테스트 실행**
- [`1:24`](https://www.youtube.com/watch?v=1_bRmkUvjHA&t=84s) **3단계 레벨 구성**: 레벨1(설치·기초 7개) → 레벨2(자동승인·플랜모드·Extended Thinking 등 중급) → 레벨3(git worktree·병렬 작업 고급 6개)
- [`2:53`](https://www.youtube.com/watch?v=1_bRmkUvjHA&t=173s) **CLAUDE.md** — 작업 지침 정의 파일. 프론트엔드/백엔드/파이썬 등 용도별 템플릿 제공
- 치트시트(명령어 1장 정리) + 23가지 팁 체크리스트 무료 제공 (설명란 링크)

> [!note]- 스크립트 주요 발언
> **0:45** "Claude Code는 프로젝트 전체를 이해하고 파일 여러 개를 동시에 수정하고 테스트까지 돌려봐요. '이거 만들어줘' 하면 진짜 다 만들어 줍니다."
>
> **1:31** "자동승인, 플랜모드, Extended Thinking — 이것만 알면 작업 속도가 훨씬 빨라져요."

## 실무 적용 포인트

| 기능 | 활용 장면 |
|------|----------|
| **Ingest (raw → wiki)** | 이 MSCL 위키의 핵심 운영 방식 |
| **CLAUDE.md** | 위키 운영 규칙·카테고리·형식 정의 (이 볼트의 `/CLAUDE.md`) |
| **Obsidian Web Clipper** | 웹 자료 → raw 폴더 자동 클리핑 |
| **플랜 모드** | 대형 작업 전 계획 수립 후 승인 |
| **Extended Thinking** | 복잡한 분석·판단이 필요한 경우 |
