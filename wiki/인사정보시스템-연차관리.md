# 인사정보 시스템 — 연차 관리

> [!tip] 서포트팀 이사 관점 핵심 takeaway
> 이 시스템은 **서면 유급휴가 신청서([[유급휴가신청서]])와 이중 운용**된다. 서면은 법적 승인 기록, 시스템은 잔여 연차·사용 이력 추적이 역할이다. 입력 누락이 생기면 수치가 엇갈리므로, 승인 후 시스템 반영을 의무화하는 프로세스 명문화가 현 단계 과제다.

---

## 시스템 개요

| 항목 | 내용 |
|------|------|
| 종류 | 사내 개발 웹 애플리케이션 (Node.js / Express) |
| 용도 | 직원 연차 발생·사용·잔여 관리 |
| 접속 | 내부 서버 또는 Render 배포 URL |
| DB | JSON 파일 기반 (`leave_db.json`) |

---

## 주요 기능

- **연차 자동 생성**: 입사 30개월 경과 시 법정 연차 자동 부여 (서버 시작 시 + 매일 UTC 00:01 실행)
- **잔여 연차 조회**: 직원별 부여·사용·잔여 일수 실시간 확인
- **반일 단위 처리**: 오전/오후 반일 사용 지원
- **직원 등록·관리**: 입사일 기준 자동 계산

| 4년 6개월 | 16일 |
| 5년 6개월 | 18일 |
| 6년 6개월~ | 20일 |

---

## 운용 주의사항

- 서면 신청서 승인 후 **시스템에 반드시 반영** (누락 시 잔여 연차 수치 불일치)
- 시스템 자동 생성은 서버 재시작 시 실행되므로 Render 환경에서 정상 작동 확인 필요
